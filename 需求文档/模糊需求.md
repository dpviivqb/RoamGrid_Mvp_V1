# MVP 

## 技术栈

```text
Next.js + TypeScript + Mapbox GL JS + Supabase + localStorage
```

## 页面必须有

```text
/
首页

/explore
探索页

/result
结果页
```

## 功能必须有

```text
1. Start Exploring
2. 定位 watchPosition
3. 实时轨迹线
4. 100m grid 解锁
5. 地图迷雾/已探索区域高亮
6. HUD：
   - 探索时间
   - 移动距离
   - 探索百分比
   - 已解锁格子数
7. Finish Exploration
8. Result 页面
9. 分享卡片
10. Supabase 保存 session / points / grids
```

---

# 一下午开发顺序

## 第 1 步：先做纯前端闭环

不要一开始接 Supabase。

先完成：

```text
首页 → 探索页 → 定位 → 轨迹 → grid解锁 → HUD → 结果页
```

数据先存在：

```text
localStorage
```

这样最快看到“开地图”的爽感。

---

## 第 2 步：再接 Supabase

只做 3 张表：

```sql
exploration_sessions
location_points
discovered_grids
```

不要做 Auth。

用户 ID 用：

```ts
anonymous_id = localStorage 生成 UUID
```

---

## 第 3 步：最后做分享卡片

用前端生成即可：

```text
html2canvas
```

截图内容包括：

```text
地图区域
轨迹
探索百分比
“我已探索城市 X%”
```

# 直接丢给 Coding Agent 的 Prompt

复制这个给 Cursor / Claude Code / Windsurf：

```text
You are building a complete Web MVP called FogCity.

Goal:
Build a Next.js + TypeScript web app that turns real-world walking into an open-world map exploration game.

Do not simplify product features. Implement all MVP features.

Tech stack:
- Next.js App Router
- TypeScript
- Tailwind CSS
- Mapbox GL JS
- Supabase JS client
- localStorage for anonymous_id
- html2canvas for share card

Pages:
1. /
Home page:
- Logo: FogCity
- Slogan: "Turn your city into an open-world game."
- Button: Start Exploring
- Clicking button navigates to /explore

2. /explore
Core page:
- Fullscreen Mapbox map
- Request geolocation permission
- Use navigator.geolocation.watchPosition
- Record location every 3-5 seconds
- Draw realtime blue route line on map
- Convert each GPS point into a grid cell
- Grid size should approximate 100m x 100m
- Mark visited grids as discovered
- Show discovered grids as bright translucent squares
- Show unexplored area as dark fog overlay effect
- HUD at top:
  - elapsed time
  - distance moved
  - exploration percentage
  - discovered grid count
- Bottom button: Finish Exploration
- On finish:
  - stop watchPosition
  - save session summary
  - navigate to /result

3. /result
Result page:
- Show:
  - distance
  - unlocked grid count
  - exploration percentage
  - duration
- Show share card:
  - title: "I explored the real world like an open-world game"
  - city exploration percentage
  - mini map screenshot or visual route summary
- Button: Download Share Card
- Button: Explore Again

Data:
Use localStorage first for reliability.
Also integrate Supabase if env variables exist.

Supabase tables:
exploration_sessions:
- id uuid primary key
- anonymous_id text
- started_at timestamptz
- ended_at timestamptz
- city_name text
- distance_meters float
- discovered_grid_count int
- exploration_percentage float

location_points:
- id uuid primary key
- session_id uuid
- lat float
- lng float
- timestamp timestamptz

discovered_grids:
- id uuid primary key
- anonymous_id text
- grid_id text
- discovered_at timestamptz

Grid algorithm:
Use an approximate meter-based grid.
Implement functions:
- getGridId(lat, lng)
- getGridPolygon(gridId)
- calculateDistance(points)
- calculateExplorationPercentage(discoveredGridCount, totalGridCount)

For MVP:
- Define exploration area as 5km x 5km around starting point
- Grid size = 100m
- Total grid count = 50 * 50 = 2500
- exploration percentage = discovered grids / 2500 * 100

Important:
- Make the UI feel like a game HUD
- Use dark translucent panels
- Use neon blue route line
- Use glowing discovered grid cells
- Keep code clean and componentized
- Add error handling for location permission denied
- Add .env.example
- Add README with setup instructions
- App must run with npm run dev
```
